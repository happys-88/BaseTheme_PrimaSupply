Write a function:

function solution(S);

that, given a string S, returns jumbled words from this string. The first character in the string will be a number. This number tells us how many words need to be created and the number of pair of words that would be available further in the string for eg:

"2, JASMEET, RAHUL, PIYUSH, ABHISHEK": would mean 2 words need to be created from this string and 2 pairs of words will be passed which are JASMEET and RAHUL as one pair and PIYUSH and ABHISHEK as the other pair.

"3, JASMEET, RAHUL, PIYUSH, ABHISHEK, ADITI, NIDHI" would mean 3 words need to be created and 3 pairs of words would be passed which are JASMEET and RAHUL as one pair, PIYUSH and ABHISHEK as the second pair and ADITI and NIDHI as the third pair.

HOW TO CREATE THE WORD

eg: "2, JAY, ROB, CAT, DOG"

There are 2 words needs to be created.

First word gets created by the pair of JAY and ROB:

  1. Compare J of JAY with R of ROB: Since J is lexicography lower than R - it becomes the first letter of the new word. WORD SO FAR: J
  2. Since J is now selected - Compare A of JAY with R of ROB, again A comes before R so A becomes the second letter in the new word. WORD SO FAR: JA
  3. Since JA are selected already - Compate Y of JAY with R of ROB, now since R comes before Y - the next letter in the new word would be R. WORD SO FAR: JAR

And then you compare Y of JAY with O of ROB and so on.

The final word in this example would be: JAROBY.

Then you move on to the second word using the pair CAT and DOG, so the function will finally return  "JAROBY, CADOGT".


For another example, given S = "2, JASMEET, RAHUL, PIYUSH, ABHISHEK" the function should return "JARAHSMEETUL, ABHIPISHEKYUSH".

You can assume that:

  • the words will only consist of uppercase English alphabet letters
  • length of the words won't exceed 100 characters

In your solution, focus on correctness. The performance of your solution will not be the focus of the assessment.
